MAC - 构建属于自己的专属博客

发表于 2017-03-25

今天通过GitHub + Node.js + HEXO + Next(主题)来建立了一个属于自己的专属博客网站。
下面就来介绍一下我的各种弯路历程吧：

首先要安装HEXO 的支持框架——Node.js ，以及版本提交工具Git

Node.js 安装

关于安装Node.js，在这里不得不好好吐槽一下，很多人写的建议是直接到官网去下载。本人就傻乎乎的真的跑到官网去下载了😢，奈何在无数次的网路连接错误后，终于放弃了😤。所以这里不得不说那些说到官网下载的人到底下载过没有😡！！！
这个Node,js的官网：https://nodejs.org/en/download/ （PS:死活下载不下来）
这个是本小主在某度找到的Node.js中国区的链接，虽然下载速度也是呵呵了，但是最起码能下载：http://nodejs.cn/download/

下载下来就是愉快的傻瓜式安装了。

Git 安装

http://git-scm.com/downloads这个同样慢的要死，除非你可以设置国外的VPN.

我建议还是用终端命令来操作，或者网上找百度云的安装包吧。（本人就是这样任性，不告诉你百度云地址😏）

还有听说安装Xcode会自带Git打死我也不会告诉你😏。

HEXO 安装

<sudo npm install -g hexo> 反正我是很快就下载好了，所以什么淘宝NPM本人就用不到了😏

初始化

<hexo init [这里是你的本地文件夹名字]>

生成静态页面

<hexo g> #g means generate

启动服务器

<hexo s> #s #means server

然后就可以在浏览器地址栏愉快的输入 http://localhost:4000

配置Github

首先你的注册呀，要不然不让你用😁

然后创建一个库，名字必须得是your_user_name.github.io（这个为什么这样写本人已经研究过了，但是不给你分享哦😏）。

然后在你的本地的建立的博客的文件夹里面找到vim_config.yml里面进行配置:

deploy:

 type: git

 repo: https://github.com/******/******.github.io.git

 branch: master

这里的**☞的就是your_user_name。

然后执行命令：
< npm install hexo-deployer-git --save >

SSH keys

< cd ~/. ssh > #检查本机已经存在的ssh密钥

< ssh-keygen -t rsa -C "邮件地址" >这里是你注册GitHub的邮箱地址哦

然后会生成一个文件 .ssh\id_rsa.pub 复制里面的ssh值 ，打开自己的GitHub的主页，找到 设置 -> SSH and GPG keys -> New SSH key

然后复制到这里：

测试是否成功

< ssh -T git@github.com > # 注意邮箱地址不用改

如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：

Hi **! You’ve successfully authenticated, but GitHub does not provide shell access.

看到这个信息说明SSH已配置成功！

此时你还需要配置：

< git config --global user.name "*****" >// 你的github用户名，非昵称
<$ git config --global user.email "*****" >// 填写你的github注册邮箱

然后使用命令
< hexo deploy > //把代码deploy到GitHub 如果你发现deploy不上可以尝试 < hexo clean >之后再deploy

然后就是重启HEXO

然后输入如下URL 你就可以把你的博客部署到GitHub了

https://******.github.io/ 你的GitHub用户名，就可以愉快的访问了。

关于vim_config.yml的配置文件怎么配置，官方给出的说明比我逼逼的更好，所以这里就给出地址大家就可以自行查看咯

https://hexo.io/zh-cn/

Next主题

如果以上你都操作成功了，那么Next主题的安装简直就小case

http://theme-next.iissnan.com/
这里给出他的官方文档，大家可以自行查看研究。

好了，这就是一个自己独立的博客安装过程。
