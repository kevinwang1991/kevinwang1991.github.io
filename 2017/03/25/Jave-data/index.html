Java 数据结构浅析

发表于 2017-03-25 |

Java.util中包含三个重要的接口以及特点：List（列表），Set（保证元素的唯一），Map（key-value键值对，保证key的唯一性），下面的图片是从CSDN网站下载的图片，是不是感觉超级复杂

List

数据增长

由于ArrayList和Vector都是的使用Object数组来进行数据存储的（因此默认生成一个新的数组的容量是10），当数据的存储大小超过默认值得时候，都会进行自动增长。这里Vector是在缺省的情况下直接扩大原来的一倍，而ArrayList则是执行一个算法：新容量=(旧容量*3)/2+1，也就是在50%左右，ArrayList总是存在着空间浪费，因此我们可以通过tirmToSize的方法在ArrayList分配完空间后去掉多余的空间。而LinkedList由于其链式存储方式，每个数组里面的对象要存储（element，next，pervious）等信息，空间的开销也是很大。

检索、插入、删除对象的效率

ArrayList和Vector都是线性存储的，因此在集合的末尾插入和删除一个对象的时间复杂度都是O（1）。但是如果在集合的其它位置来进行添加和删除的时间复杂度就会程线性增长O（n-i），因为你在插入和删除对象的同时，元素都要进行位移操作。
而LinkedList由于是双向链式存储，因此在存储每个对象都会记录改对象的上一个元素和下一个元素,因此在插入和删除一个元素，它的时间复杂度是O（1）。
而在随机检索方面，由于线性列表的缘故，ArrayList和Vector的get（）方法可以得到任何一个对象，因此他们的时间复杂是一个固定的值，并且要远远小于LinkedList。而LinkedList做随机检索通过其get（）方法其实是在做顺序查找，其时间复杂度跟list的大小成正比的。

总结

  • ArrayList和LinkedList是线程不安全的，Vector是线程安全的。
  • 在集合的末端添加、一处元素，Vector和ArrayList都可以，但是如果在集合中间某个位置进行同样的操作，LinkedList执行效率更高。
  • 在进行随机检索的时候LinkedList的执行效率比ArrayList和Vector要低。
  • 在空间开销方面，Vector和ArrayList比，Vector比ArrayList要大，但是都可以通过方法控制大小。LinkedList的开销主要在分配一个内部的Entry对象。

Set

HashSet

HashSet继承了Java.util.set接口，具有以下特点：

  • 不允许出现重复的元素,如果插入同样的元素，则会发生覆盖
  • 不能保证元素的顺序
  • 允许包含为null的元素，但是最多只能有一个null的元素
  • 不是线程同步的

当向HashSet中插入一个元素的时候，HashSet会调用该对象的hashCode（）方法来得到该对象的hashCode值，然后根据hashCode的值来决定该对象在HashSet中存储位置。
HashSet集合判断两个元素相等时通过equals方法来比较的，并且两个对象的hashCode（）方法返回的值相等。

TreeSet

TreeSet是可以实现排序功能的集合，它可以使对象在加入集合的过程中按照某种比较规则进行排序之后再添加。

那么TreeSet是怎么进行排序呢？TreeSet支持两种排序方式：

  • 自然排序
  • 客户化排序

说道自然排序就得提到JDK中的comapreTo（Object o）方法（默认是升序排序），改方法是继承了Comparable接口的类都可以实现的方法。（比如Integer，Double，String等）。该方法返回整数类型，对于表达式x.compareTo(y)，如果返回0则表示x和y相同，如果返回值大于0，则表示x大于y，如果小于0，则表示x小于y。

客户化排序：
java.util.Comparator接口用于指定具体的排序方式，具体的语法是compare(Object obj1,Object obj2)，该表达式返回的值得意义大小跟x.compareTo(y)是一样的。

LinkedHashSet

LinkedHashSet跟HashCSet存储方式一样，但是它同时使用链式方式来维护元素的次序。因此在遍历元素的时候LinkedHashSet的效率比HashSet要高，但是在插入元素的时候由于会记录元素的前后元素地址，因此性能没有HashSet好。

Map

HashMap

HashMap是最长用的Map，它根据键的HashCode值存储数据，根据Key可以直接获得它的value，具有以下几个特点：

  • 具有很快的访问速度
  • 遍历时，去的数据的顺序是完全随机的
  • 最多只允许一条记录的Key为null
  • 允许多条记录的值为null
  • 不支持线程同步，如果需要同步，可以使用Collection的synchronizedMap方法使HashMap具有同步能力
  • 具有containsValue和containsKey方法
  • 通过大量的统计，默认的初始值大小为16，负载因子为0.75（这里通过增大默认的负载因子来控制每次自动增大的空间，但是增大负载因子会减小Hahs表在空间所占的内存空间，相应的却会增大查询数据的时间开销）

HashTable

HashTable和HashMap的存储方式是一样的，但是它有别于HashMap的地方在于：

  • 方法是同步的
  • 方法不允许value = null
  • 继承自Dictionary类
  • 由于都是采用Hash算法，因此性能和HashMap差不多，但是由于采用了线程同步，可能存取速度比HashMap稍差
  • 只有contains方法
  • 默认size初始值为11，扩容因子为0.75（如果到达最大默认size，扩容后的大小为原来的大小的2倍+1）

TreeMap

TreeMap是一个有序的key-value集合，是通过红黑二叉树来实现的（关于红黑二叉树需要另起文章研究）

  • 由于采用的红黑二叉树的排序方式，因此在存入和取出元素的效率都比HashMap要低。
