Java.util中包含三个重要的接口以及特点：List（列表），Set（保证元素的唯一），Map（key-value键值对，保证key的唯一性），下面的图片是从CSDN网站下载的图片，是不是感觉超级复杂

List

同步性

ArrayList和LinkedList不是线程同步的，而Vector是线程安全的，因此在不是多线程的情况下使用ArrayList和LinkedList效率会大大提升。不过如果想在多线程的情况下使用ArrayList和LinkedList，可通过如下方法来实现：

但是执行的效率并没有Vector高。

数据增长

由于ArrayList和Vector都是的使用Object数组来进行数据存储的（因此默认生成一个新的数组的容量是10），当数据的存储大小超过默认值得时候，都会进行自动增长。这里Vector是在缺省的情况下直接扩大原来的一倍，而ArrayList则是执行一个算法：新容量=(旧容量*3)/2+1，也就是在50%左右，ArrayList总是存在着空间浪费，因此我们可以通过tirmToSize的方法在ArrayList分配完空间后去掉多余的空间。而LinkedList由于其链式存储方式，每个数组里面的对象要存储（element，next，pervious）等信息，空间的开销也是很大。

检索、插入、删除对象的效率

ArrayList和Vector都是线性存储的，因此在集合的末尾插入和删除一个对象的时间复杂度都是O（1）。但是如果在集合的其它位置来进行添加和删除的时间复杂度就会程线性增长O（n-i），因为你在插入和删除对象的同时，元素都要进行位移操作。
而LinkedList由于是双向链式存储，因此在存储每个对象都会记录改对象的上一个元素和下一个元素,因此在插入和删除一个元素，它的时间复杂度是O（1）。
而在随机检索方面，由于线性列表的缘故，ArrayList和Vector的get（）方法可以得到任何一个对象，因此他们的时间复杂是一个固定的值，并且要远远小于LinkedList。而LinkedList做随机检索通过其get（）方法其实是在做顺序查找，其时间复杂度跟list的大小成正比的。

总结

  • ArrayList和LinkedList是线程不安全的，Vector是线程安全的。
  • 在集合的末端添加、一处元素，Vector和ArrayList都可以，但是如果在集合中间某个位置进行同样的操作，LinkedList执行效率更高。
  • 在进行随机检索的时候LinkedList的执行效率比ArrayList和Vector要低。
  • 在空间开销方面，Vector和ArrayList比，Vector比ArrayList要大，但是都可以通过方法控制大小。LinkedList的开销主要在分配一个内部的Entry对象。

Set

HashSet

HashSet继承了Java.util.set接口，具有以下特点：

  • 不允许出现重复的元素
  • 不能保证元素的顺序
  • 允许包含为null的元素，但是最多只能有一个null的元素

TreeSet

TreeSet是可以实现排序功能的集合，它可以使对象在加入集合的过程中按照某种比较规则进行排序之后再添加。

那么TreeSet是怎么进行排序呢？TreeSet支持两种排序方式：

  • 自然排序
  • 客户化排序

说道自然排序就得提到JDK中的comapreTo（Object o）方法，改方法是继承了Comparable接口的类都可以实现的方法。（比如Integer，Double，String等）。该方法返回整数类型，对于表达式x.compareTo(y)，如果返回0则表示x和y相同，如果返回值大于0，则表示x大于y，如果小于0，则表示x小于y。

客户化排序：
java.util.Comparator接口用于指定具体的排序方式，具体的语法是compare(Object obj1,Object obj2)，该表达式返回的值得意义大小跟x.compareTo(y)是一样的。

MAC - 构建属于自己的专属博客

发表于 2017-03-25 |

今天通过GitHub + Node.js + HEXO + Next(主题)来建立了一个属于自己的专属博客网站。
下面就来介绍一下我的各种弯路历程吧：

首先要安装HEXO 的支持框架——Node.js ，以及版本提交工具Git

Node.js 安装

关于安装Node.js，在这里不得不好好吐槽一下，很多人写的建议是直接到官网去下载。本人就傻乎乎的真的跑到官网去下载了😢，奈何在无数次的网路连接错误后，终于放弃了😤。所以这里不得不说那些说到官网下载的人到底下载过没有😡！！！
这个Node,js的官网：https://nodejs.org/en/download/ （PS:死活下载不下来）
这个是本小主在某度找到的Node.js中国区的链接，虽然下载速度也是呵呵了，但是最起码能下载：http://nodejs.cn/download/

下载下来就是愉快的傻瓜式安装了。

Git 安装

http://git-scm.com/downloads这个同样慢的要死，除非你可以设置国外的VPN.

我建议还是用终端命令来操作，或者网上找百度云的安装包吧。（本人就是这样任性，不告诉你百度云地址😏）

还有听说安装Xcode会自带Git打死我也不会告诉你😏。

HEXO 安装

<sudo npm install -g hexo> 反正我是很快就下载好了，所以什么淘宝NPM本人就用不到了😏

初始化

<hexo init [这里是你的本地文件夹名字]>

生成静态页面

<hexo g> #g means generate

启动服务器

<hexo s> #s #means server

然后就可以在浏览器地址栏愉快的输入 http://localhost:4000

配置Github

首先你的注册呀，要不然不让你用😁

然后创建一个库，名字必须得是your_user_name.github.io（这个为什么这样写本人已经研究过了，但是不给你分享哦😏）。

然后在你的本地的建立的博客的文件夹里面找到vim_config.yml里面进行配置:

deploy:

 type: git

 repo: https://github.com/******/******.github.io.git

 branch: master

这里的**☞的就是your_user_name。

然后执行命令：
< npm install hexo-deployer-git --save >

SSH keys

< cd ~/. ssh > #检查本机已经存在的ssh密钥

< ssh-keygen -t rsa -C "邮件地址" >这里是你注册GitHub的邮箱地址哦

然后会生成一个文件 .ssh\id_rsa.pub 复制里面的ssh值 ，打开自己的GitHub的主页，找到 设置 -> SSH and GPG keys -> New SSH key

然后复制到这里：

测试是否成功

< ssh -T git@github.com > # 注意邮箱地址不用改

如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：

Hi **! You’ve successfully authenticated, but GitHub does not provide shell access.

看到这个信息说明SSH已配置成功！

此时你还需要配置：

< git config --global user.name "*****" >// 你的github用户名，非昵称
<$ git config --global user.email "*****" >// 填写你的github注册邮箱

然后使用命令
< hexo deploy > //把代码deploy到GitHub 如果你发现deploy不上可以尝试 < hexo clean >之后再deploy

然后就是重启HEXO

然后输入如下URL 你就可以把你的博客部署到GitHub了

https://******.github.io/ 你的GitHub用户名，就可以愉快的访问了。

关于vim_config.yml的配置文件怎么配置，官方给出的说明比我逼逼的更好，所以这里就给出地址大家就可以自行查看咯

https://hexo.io/zh-cn/

如果以上你都操作成功了，那么Next主题的安装简直就小case
